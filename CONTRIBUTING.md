### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug in detail and provide steps to reproduce it.
- Include information about your device and environment (Expo version, Android/iOS version).

### Suggesting Enhancements
- Check if your idea has already been suggested.
- Open an issue with the [Enhancement] tag.
- Explain why this feature would be useful for sales workflows.

### Pull Requests
- Ensure your code follows the existing style (TypeScript + NativeWind).
- Update the documentation if you're adding new features.
- Test your changes thoroughly on a real device or emulator.

Sales Expert follows a specific aesthetic:
- **Primary Colors**: Black (`#000000`) and Gold (`#FFC107`).
- **Typography**: Tracking should be tight (`tracking-tight`) for headings and wide (`tracking-widest`) for uppercase labels.
- **Components**: UI should feel "Premium" and "Glassmorphic" (use `bg-white/5` with `border-white/5`).
